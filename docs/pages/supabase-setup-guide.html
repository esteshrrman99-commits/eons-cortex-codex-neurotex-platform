Project setup guide and SQL checklist

This page explains how to create the Supabase project, run the draft SQL, create the private bucket, and keep secrets out of Git and chat.

Secrets: Outside GitSQL: Draft ReviewUpload: Still Blocked

Setup Steps

1. Create Supabase project in the Supabase dashboard.
2. Open SQL Editor.
3. Review and run supabase/sql/001_eons_schema_draft.sql.
4. Confirm tables exist.
5. Review RLS draft before relying on it.
6. Create private bucket: eons-evidence-vault.
7. Keep service role key out of GitHub Pages, Git, docs, screenshots, and chat.
8. Add secrets only to the private backend runtime environment later.
9. Test with fake records only.
10. Keep sensitive production upload blocked until security and professional review.

Private Bucket

Bucket: eons-evidence-vault
Public access: off
Public URLs: disabled for sensitive evidence
Uploads: backend only after auth, encryption, audit log, CORS, backup, and reviewer gates pass

Secret Boundary

Never expose:
- SUPABASE_SERVICE_ROLE_KEY
- database password
- JWT secret
- storage signing secrets
- backup credentials

Allowed later only in backend runtime secrets:
- SUPABASE_URL
- SUPABASE_SERVICE_ROLE_KEY
- EONS_CORS_ORIGIN
