Pass / Fail

PASS:
Fake records move through approved backend flow, RLS blocks unauthorized access, private bucket is locked, audit events are recorded.

FAIL:
Any real sensitive data is used, any key appears in Git/browser/chat, public bucket access is enabled, RLS is disabled, or upload bypasses the backend gate.
