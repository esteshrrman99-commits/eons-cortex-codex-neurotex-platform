GitHub Pages Can Safely Handle

- public/static interface
- proof reports

GitHub Pages Cannot Be

- private encrypted database
- runtime secret manager
- sensitive production file vault
- backend authentication server
- private audit-log database
- professional review authority

Real Compliance Gates Required

Auth: private backend login required
Encryption: server-side encryption required
Database: private database required
Private storage: private encrypted bucket required
Secrets: provider/GitHub Actions secrets only
Backup: encrypted backup + restore test required
Audit logging: backend append-only events required
CORS: backend locked to approved origin
Reviewer approval: CPA/EA/tax attorney workflow required

- real sensitive production uploads
- secrets
- private database

Sensitive upload must stay blocked until the private backend passes all real gates.

Boundary
